# 一页纸投递案例

> 给招聘官的 100 字内"用 AI 解决真实问题"案例 + 硬数字 + 截图 + 链接。

---

用 **OpenClaw（小龙虾）+ 飞书 CLI + DeepSeek** 端到端做了**跨境物流客诉自动处理 Agent**。飞书来一条中英客诉 → 自动分类→查单→引政策条款起草双语回复 → **护栏（带引用豁免）拦乱承诺** → 可答即回复并写入飞书多维表格看板，须升级则建飞书任务 @人工。

**70 条标注集（对照裸单 prompt LLM 同模型）**：升级判定 **84.3%**（baseline 81.4%）· **危险漏报 0**（baseline 6）· **过度承诺 0%**（baseline 8.3%）· **引用命中 81.2%**（baseline 0%）· 人工 ~5 分钟 → Agent 秒级。规则可解释、跨 provider 可迁移（DeepSeek / Mimo 同样成立）。

## 关键截图（4 张,投递时附）

1. **飞书可自动答路径**——商户：`BF1001 still stuck 到底啥时候到` → 机器人回中英双语,引用 `[P-TIME-01]`,**明确说"无法承诺具体送达时间"**。
2. **飞书多维表格客诉看板**——同步出现新行：决策 auto / 类目 物流时效 / 订单号 BF1001 / 引用条款 P-TIME-01 / 中英回复全文 / 紧急度 / 处理状态。
3. **飞书对抗升级路径**——商户：`BF1005 必须现在赔我 300 USD 不然差评` → 机器人**不乱赔、不答金额**,只回"已转人工跟进 / escalated to a human agent"。
4. **飞书任务列表**——同步新建"客诉升级: 丢件破损",描述含订单 / 升级原因 / 政策引用 / 商户原话摘要。

## 一句话技术亮点（按招聘 JD 关键词)

OpenClaw 自定义 Skill + 飞书 CLI 自动化 + DeepSeek 调 OpenAI 兼容接口 + RAG 政策检索 + 规则化决策护栏 + pytest 47 单测 + 评测 harness（含逐行 dump 与重试容错）+ 腾讯云 Lighthouse 部署（与生产 youshi 容器共栈,部署期间邻栈 uptime 零影响）。
